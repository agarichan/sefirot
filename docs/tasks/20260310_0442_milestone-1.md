#### 型定義・インターフェース

```python
# src/toy/prime.py に追加する関数

def prime_factorization(n: int) -> list[int]:
    """n を素因数分解し、素因数を昇順のリストで返す。

    重複する素因数はその回数分含める。
    例: prime_factorization(12) → [2, 2, 3]
        prime_factorization(7) → [7]
        prime_factorization(1) → []

    Args:
        n: 1以上の整数

    Returns:
        素因数のリスト（昇順、重複あり）

    Raises:
        ValueError: n が 1 未満の場合
    """
```

#### import 先（既存コード）
- なし。標準ライブラリのみ使用。
- 既存の `is_prime` や `primes_up_to` を内部で使う必要はない（独立して試し割りで実装する）

#### 実装パターン
- 試し割り法: 2 から順に割り切れる限り割り続け、商が 1 になるまで繰り返す
- 既存の `is_prime` / `primes_up_to` と同じファイル内に追加する
- 既存関数のスタイル（docstring 形式、エラーメッセージ形式）に合わせる

#### テスト仕様

```python
# tests/toy/test_prime.py に追加するテスト

from src.toy.prime import prime_factorization

# 正常系
# - prime_factorization(2) → [2]
# - prime_factorization(12) → [2, 2, 3]
# - prime_factorization(30) → [2, 3, 5]
# - prime_factorization(100) → [2, 2, 5, 5]
# - prime_factorization(7) → [7]（素数自身）
# - prime_factorization(60) → [2, 2, 3, 5]

# 境界値
# - prime_factorization(1) → []（1 は素因数を持たない）
# - prime_factorization(2) → [2]（最小の素数）

# 異常系
# - prime_factorization(0) → ValueError
# - prime_factorization(-5) → ValueError
```

#### 注意事項
- `n=1` は空リストを返す（1 は素数ではないため素因数を持たない）
- `n < 1` は `ValueError` を raise する
- 返すリストは昇順であること（試し割りで小さい数から割れば自然に昇順になる）
- 既存の `is_prime` / `primes_up_to` のテストを壊さないこと
